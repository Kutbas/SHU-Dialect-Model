# Live2D 接入更新说明

## 1. 和最初拿到项目时相比，发生了什么变化

原始仓库 `SHU-Dialect-Model-main` 本身已经有：

- FastAPI 后端
- 已打包好的前端页面
- 小沪主题聊天界面

但最初并没有真正把 Live2D 数字人接到主页面里，也没有数字人状态切换、互动气泡这些能力。

当前版本相比原始项目，已经新增了下面这些能力：

### 页面层

- 小沪主题页面右侧增加了独立数字人展示区域
- 输入框和聊天内容区域为数字人让出了空间，避免严重遮挡
- 数字人能在主页面稳定显示，不再只是单独 sample 演示

### 资源层

- 已把运行时需要的 Live2D 资源放进仓库
- 当前真正用于运行的是：
  - `www/live2d-widget/`
  - `www/live2d-widget/Resources/xiaohu-states/`
- 这些资源已经包含：
  - `loading`
  - `speaking`
  - `yes`
  - `deny`

### 状态层

- 默认待机状态为 `loading`
- 发送消息时切换到 `loading`
- 音频播放时切换到 `speaking`
- 音频结束后短暂切到 `yes`

### 交互层

- 点击小沪人物模型，可以弹出上海话互动气泡
- 互动语目前来自我们整理过的上海话短句
- 气泡目前显示在人物上方，适合演示

## 2. 本次主要改动文件

### 主页面宿主层

- `www/index.html`
- `www/live2d-host.js`
- `www/live2d-host.css`

作用：

- 把 Live2D widget 挂到主页面
- 管理显示/隐藏、状态联动、互动气泡

### Live2D 运行资源

- `www/live2d-widget/`

作用：

- 存放可直接运行的 Live2D widget 构建产物
- 存放运行时模型资源、Core、Shader、状态资源

### 当前关键状态资源目录

- `www/live2d-widget/Resources/xiaohu-states/manifest.json`
- `www/live2d-widget/Resources/xiaohu-states/loading/`
- `www/live2d-widget/Resources/xiaohu-states/speaking/`
- `www/live2d-widget/Resources/xiaohu-states/yes/`
- `www/live2d-widget/Resources/xiaohu-states/deny/`

## 3. 当前版本可以演示什么

明天给老师演示时，当前版本可以稳定展示这些点：

1. 进入小沪主题后，右侧会显示数字人
2. 默认待机状态是 `loading`
3. 发送消息、语音播放、播放结束时，小沪会切不同状态
4. 点击数字人，会弹出上海话互动气泡

## 4. 目前仍然存在的限制

这部分建议演示时不用主动展开太多，但自己要心里有数：

- 状态切换已经比最初顺很多，但还不是完全无缝
- 现在本质上还是“同角色多状态模型切换”，不是“同一个底模只切 motion”
- `yes / deny` 还不是根据语义自动判断的，而是目前先做了基础联动

## 5. 为什么别人 clone 之后也能继续正常使用

当前主项目运行时依赖的数字人资源已经在仓库里，而不是只放在你本地素材目录里。

也就是说，别人 clone 这个仓库后，只要：

1. 安装项目原本的 Python 依赖
2. 正常启动 FastAPI 服务

就可以直接使用当前这版数字人功能。

### 已确认的关键点

- 当前主页面引用的是仓库内相对路径：
  - `/live2d-host.css`
  - `/live2d-host.js`
  - `/live2d-widget/index.html`
- 没有保留指向你本机 `D:\Study_Main\Xiao_Hu\...` 这类绝对路径的运行时依赖
- 运行所需的模型资源已经复制进 `www/live2d-widget/Resources/`

## 6. 如果以后还要继续改，建议怎么做

建议沿用现在这套流程：

1. 先备份稳定版
2. 在 SDK sample 或隔离环境中试改
3. build 成新的 widget
4. 再同步回 `www/live2d-widget/`
5. 最后只在主页面宿主层做轻量联动

## 7. 这次最值得保留的成果

- 数字人已经真正接入主项目，而不是停留在外部 demo
- 多状态运行链路已经跑通
- 默认 `loading` 待机已经稳定
- 点击数字人互动也已经有演示效果
- 项目已经具备继续做“更智能、更自然数字人”的基础
